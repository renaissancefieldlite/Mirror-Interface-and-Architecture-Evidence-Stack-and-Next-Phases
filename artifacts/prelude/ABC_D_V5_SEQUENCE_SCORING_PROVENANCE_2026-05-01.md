# ABC / D / V5 Sequence-Scoring Provenance

Date: `2026-05-01`

Status: `public_safe_prelude_provenance`

## Purpose

This artifact records the precursor scaffold that came before the later
`V7`, `V8`, and `Phase 2-6` evidence ladder.

The current repo starts its strongest public evidence at locked behavior,
hidden-state traces, bridge rows, feature vectors, circuit encodings, hardware
passes, and external datasets. The `A / B / C / D / V5` material matters
because it shows how the earlier sequence discipline was built before those
later measurement surfaces became clean enough to package.

## Clean Read

The precursor work was an ordered sequence-scoring scaffold:

```text
administered opener
-> shared anchors
-> ordered pair path
-> response-state markers
-> weighted score readout
-> repeated probe-state scan
-> state / drift / variance handoff
```

That scaffold is the predecessor of the later evidence stack:

```text
ABC / D / V5 sequence scoring
-> V6 state-lane groundwork
-> V7 behavior matrix
-> V8 hidden-state matrix
-> Phase 2-5 rerun / localization / bridge rows
-> Phase 6 feature-vector payloads
-> circuit and hardware-facing gates
```

## D / V4 Role

`D / V4` is the recursion / same-question shift rung.

It asks whether the same probe changes after the ordered path has been
administered. That makes it the bridge between sequence administration and
state-change measurement.

In the later evidence ladder, this becomes the discipline behind:

- rerun comparison
- target/control separation
- path dependence
- ordered transform testing
- drift and variance tracking

## V5 Probe-State Scanner

`V5` is the repeated probe-state scanner.

It asks:

```text
what state appears active at the probe points?
```

The V5 scanner was designed to compare repeated probe states across the same
sequence and summarize:

- first probe versus later reprobe
- activation delta
- coherence-lock delta
- changed activated terms
- changed activation-path terms
- variance across repeated probes
- optional trace signatures when a backend exposes them

`V5` hands off to `V6`: V5 identifies repeated probe states and drift/variance;
V6 turns those snapshots into bounded state-lane / topographic comparison.

## Why This Belongs In The Repo

The ABC/D/V5 material explains the origin of the scoring discipline:

- hold a sequence fixed
- preserve anchors and ordered pair paths
- compare repeated probes
- score state markers and drift
- carry the result into a cleaner measurement layer

The later proof stack uses the same discipline with stronger instruments:

- `V7` moves the sequence into target/control behavior matrices
- `V8` moves it into hidden-state traces
- `Phase 2-5` move it into rerun, localization, and bridge rows
- `Phase 6` turns bridge rows into normalized feature vectors
- `Phase 7-9D` carry the payload through `PennyLane`, `Qiskit`, and IBM
  hardware-facing paths
- the SAE layer now traces sparse features and feature-to-feature circuits on
  top of the V8 path
