# Shared Architecture Spine

Date: `2026-04-24`

## Core Claim

The shared claim is not that one finished quantum product already solves all
five enterprise problems.

The stronger and cleaner claim is:

the same state-control-transform-invariant-drift-score architecture can be
used to build disciplined local predecessor models, then fork those models
into quantum-enabled PoC candidates for each enterprise surface.

## Shared Schema

Every track uses:

```text
state / control / transform / invariant / drift / coherence / score
```

## Shared Demonstrator

The Lattice Model Node Companion is the visual front end.

It should show:

- the enterprise state object
- the relevant nest
- the local predecessor engine
- the quantum-enabled fork
- the validation metric
- the failure / drift mode
- the PoC output

## Shared Proposal Skeleton

Each proposal should answer:

1. What is the enterprise state object?
2. What is the baseline or null control?
3. What transform or intervention is tested?
4. What invariant must be preserved?
5. What drift mode matters commercially or technically?
6. What score decides improvement?
7. What local predecessor model exists before quantum?
8. What quantum-enabled method is being tested?
9. What benchmark decides success?
10. What does the Phase II PoC produce?

## Build Philosophy

Under-promise externally.

Over-deliver in the local demonstrator.

The campaign should feel like:

one architecture, five serious technical doors.
