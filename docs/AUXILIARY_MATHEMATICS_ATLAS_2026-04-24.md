# Auxiliary Mathematics Atlas

Date: `2026-04-24`

Status:
Nest `1` expansion map

## Purpose

`Nest 1` started with the smallest formal substrate:

- linear algebra
- symmetries
- invariants
- encoded circuit states

That base is correct, but it is not the whole mathematical support layer.

This atlas records the auxiliary mathematical disciplines that should be
available as the nesting ladder expands into matter, dynamics, biology,
enterprise systems, and cosmic-scale structure.

## Minimal Base vs Auxiliary Base

The minimal base remains:

```text
linear algebra + symmetry + invariants + encoded states
```

The expanded base becomes:

```text
representation
+ relation
+ transform
+ uncertainty
+ dynamics
+ optimization
+ control
+ implementation stability
```

## Why This Matters

The higher nests will need different mathematical lenses:

- `Nest 2` matter needs graph theory, geometry, group theory, topology,
  topography, spectral methods, and numerical computation
- `Nest 3` dynamics needs differential equations, dynamical systems, control
  theory, spectral analysis, probability, and numerical stability
- `Nest 4` biology needs statistics, information theory, network theory,
  dynamical systems, and control
- `Nest 5` convergence needs graph theory, optimization, game / decision
  theory, systems dynamics, topology, and statistics

## Comparator Rule

Every auxiliary discipline must still reduce back into the shared schema:

`state / control / transform / invariant / drift / coherence / score`

If a math lens cannot define those fields, it is not ready to become a build
engine.
