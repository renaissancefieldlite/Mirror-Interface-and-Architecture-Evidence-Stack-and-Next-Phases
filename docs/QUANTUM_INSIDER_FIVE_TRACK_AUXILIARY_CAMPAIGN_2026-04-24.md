## Purpose

The Lattice Model Node Companion creates a useful new possibility:

one shared local demonstration chassis that can be forked into multiple
challenge-specific proposal tracks.

Internal challenge posture:

build predecessor models for all five enterprise tracks.

External posture:

submit only disciplined, track-specific proposals where the local engine,
dataset path, validation story, and quantum-enabled angle are credible.

The point is not to spray one generic pitch across five sponsors. The point is
to use the same Mirror Architecture spine and companion demo as the shared
interface, then tailor the engine, scoring layer, and proof path to each
enterprise need.

## Track-Specific Build Notes

### Airbus

Best local predecessor:

- build `Nest 3` oscillator / field / flow bounded model first
- add simple aerodynamic-state language:
  pressure, velocity, boundary, vortex, turbulence, lift / drag proxy
- show how preserved structure and drift penalties behave under perturbation
- later map to PhysicsNeMo or a classical CFD surrogate before any quantum
  claim

### Cleveland Clinic

Best local predecessor:

- build `Nest 2 -> Nest 4` protein / allostery graph comparator
- state object:
  residue graph, binding site, allosteric pathway, conformational state
- invariant:
  conserved fold / pathway relation / active-site communication
- drift:
  false pathway, misfold, off-target effect, unstable conformer
- later quantum lane:
  quantum simulation candidate for small constrained pathway fragments

### E.ON

Best local predecessor:

- build graph planning engine from the existing lattice score schema
- state object:
  grid nodes, loads, constraints, capacity, reliability windows
- invariant:
  service continuity, capacity balance, resilience, cost constraint
- drift:
  bottleneck, outage exposure, infeasible expansion, overload propagation
- later quantum lane:
  hybrid optimization on graph expansion scenarios

### HSBC

Best local predecessor:

- build transaction graph anomaly model
- state object:
  user, merchant, card, device, timing, amount, channel, sequence
- invariant:
  normal behavioral lattice and risk envelope
- drift:
  fraud-like deviation, synthetic pattern, account takeover, false positive
- later quantum lane:
  quantum kernel / hybrid feature-map benchmark against classical controls

### Volkswagen Group Innovation

Best local predecessor:

- use the existing AI-side evidence as the strongest bridge:
  V7 behavioral control separation and V8 hidden-state separation
- state object:
  scene, instruction, perception token set, action plan, control route
- invariant:
  stable perception-action relation and safety-preserving plan
- drift:
  hallucinated object, unsafe plan, action mismatch, multimodal routing failure
- later quantum lane:
  representation search, route optimization, and action-policy stability
  benchmark

## Shared Proposal Architecture

Each proposal should share a common skeleton:

1. define enterprise state object
2. define control / null baseline
3. define transform / intervention
4. define invariant / coherence target
5. define drift / failure mode
6. define score
7. show local predecessor engine
8. define quantum-enabled fork
9. define validation benchmark
10. define PoC output

## Campaign Strategy

Internal:

build all five predecessor engines as a challenge to make the architecture
legible across every track.

External:

lead with the strongest one or two proposals first, then reuse the shared
chassis to support additional applications only if the proof path is crisp.

The strongest near-term tracks appear to be:

- `E.ON`, because grid expansion maps naturally to graph optimization
- `Volkswagen`, because V7/V8 already support AI control / hidden-state claims
- `Cleveland Clinic`, because the existing `Nest 2 -> Nest 4` biology and
  allostery mapping can become a strong research story if bounded carefully

Airbus and HSBC remain viable if the local predecessor engines are built
clearly enough before submission.
